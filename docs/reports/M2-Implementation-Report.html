Testing vs Production

Understanding how the A²E routing works in testing mode vs production integration.

The Key Concept: hasInternalDemand

Important Understanding

The hasInternalDemand parameter is NOT determined by A²E Engine. In production, TokenOS tells A²E whether there's internal demand based on whether an agent task is waiting. The dashboard toggle simulates both scenarios for testing purposes.

Comparison: Testing vs Production

Testing Mode (Dashboard)
  • You manually select "Internal Demand" toggle
  • Simulates TokenOS sending different requests
  • Allows testing both routing scenarios
  • Dashboard calls POST /v1/route directly
  • Results displayed in real-time
Production Mode (TokenOS)
  • TokenOS checks if agent task is waiting
  • TokenOS sets hasInternalDemand automatically
  • A²E receives request, makes decision
  • A²E returns: market, rate, reason
  • TokenOS routes job to selected market

Production Integration Flow

1
GPU Node Comes Online
Node agent registers with A²E: POST /v1/nodes
↓
2
TokenOS Checks for Work
Platform checks if any agent tasks need GPU compute
↓
3
TokenOS Calls A²E
POST /v1/route with { gpuTier, deploymentId, hasInternalDemand: true/false }
↓
4
A²E Makes Decision
Returns: { market: "INTERNAL" or "AKASH", rate, reason }
↓
5
TokenOS Executes
Routes job to selected market and tracks earnings

What the Dashboard Toggle Simulates

Toggle Setting Simulates Expected Result
"No (Route to external)" GPU is idle, no agent tasks waiting Routes to AKASH or IONET (highest paying)
"Yes (Premium rate)" Agent task waiting for GPU Routes to INTERNAL at retail rate

Routing Scenarios

Test cases demonstrating the A²E routing decision engine.

Scenario 1: Internal Demand (Premium Rate)

Input Value
GPU TierH100
hasInternalDemandtrue
Expected Result

Market: INTERNAL | Rate: $140.15/day | Reason: Internal demand — routing to INTERNAL at retail rate

Scenario 2: No Internal Demand (External Market)

Input Value
GPU TierH100
hasInternalDemandfalse
Expected Result

Market: AKASH | Rate: ~$91/day | Reason: No internal demand — routing to AKASH ($91.10/day)

Scenario 3: Yield Floor Applied

Input Value
GPU TierH100
hasInternalDemandfalse
External Market Rate$70/day (below $83 floor)
Expected Result

Market: AKASH | Rate: $83/day (boosted) | yieldFloorApplied: true | Reason: Yield floor applied — rate boosted from $70 to $83/day

Scenario 4: Different GPU Tiers

GPU Tier Internal Rate External Rate (Akash) Cost Floor
H100 $140.15/day ~$91/day $83/day
H200 $179.85/day ~$115/day $105/day
B200 $321.10/day ~$200/day $170/day
B300 $431.75/day ~$280/day $250/day
GB300 $499.35/day ~$350/day $300/day

Decision Logic Summary

1
Check Internal Demand
If hasInternalDemand = true → INTERNAL market (premium)
↓ No internal demand
2
Compare External Markets
Check Akash and IO.net rates, select highest paying
↓
3
Apply Yield Floor
If external rate < floor → boost to floor rate

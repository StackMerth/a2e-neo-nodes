Project Overview

The A²E (Arbitrage & Orchestration Engine) is a standalone microservice that optimizes GPU node earnings for the TokenOS compute platform. It intelligently routes jobs between internal demand and external markets (Akash Network, IO.net) to maximize revenue while guaranteeing a yield floor for node operators.

A²E Routing Logic

Core arbitrage engine decision flow and market routing strategy.

Decision Flow

1
Route Request Received
POST /v1/route with gpuTier, deploymentId, hasInternalDemand
↓
2
Check Internal Demand
If internal demand exists, route to INTERNAL market (premium rate)
↓
3
Compare External Markets
Fetch current rates from Akash Network and IO.net
↓
4
Select Highest Paying Market
Rank external markets by rate per hour for GPU tier
↓
5
Apply Yield Floor
Ensure rate meets minimum guaranteed yield for tier
↓
6
Return Decision + Log
Return market, rate, reason; log to RoutingLog table

Rate Priority Logic

Priority Market Rate Type When Used
1 INTERNAL Retail Rate When hasInternalDemand = true
2 AKASH / IONET Wholesale Rate Highest paying external market
3 YIELD_FLOOR Cost Floor If external rates below minimum

Example Request/Response

Request

POST /v1/route Content-Type: application/json X-API-Key: your-api-key { "deploymentId": "#104", "gpuTier": "H100", "hasInternalDemand": false }

Response

{ "market": "AKASH", "ratePerHour": 5.25, "ratePerDay": 126.00, "reason": "No internal demand — routing to AKASH", "yieldFloorApplied": false, "timestamp": "2026-03-22T10:30:00Z" }

Rate Caching Strategy

Redis Cache Configuration

External market rates are fetched every 60 seconds and cached in Redis with a 60-second TTL. Historical rates are stored in PostgreSQL (MarketRateHistory table) for analytics and trend analysis. If an external API fails, the system gracefully falls back to the last cached rate.

API Key Authentication

All endpoints require authentication via the X-API-Key header. API keys are stored securely as environment variables and validated on each request. Rate limiting is applied per API key to prevent abuse.

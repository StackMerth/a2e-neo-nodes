# A²E Engine - Claude Code Instructions

## Project Overview
**Client:** Emmanuel (TokenOS)
**Platform:** compute.tokenos.ai
**Project:** A²E (Arbitrage & Orchestration Engine) for decentralized GPU compute

## Critical Rules

### Code Quality Standards
- All code must be **production-ready** — no prototype-quality, no "good enough for now"
- Follow industry best practices for Node.js/TypeScript development
- Proper error handling with meaningful error messages
- Input validation on all API endpoints
- No hardcoded values — use environment variables and config files
- Write clean, readable, self-documenting code
- Use meaningful variable and function names
- Keep functions small and single-purpose
- Add JSDoc comments for public APIs and complex logic only

### Git Commit Rules
- **NEVER** include "Co-Authored-By: Claude" or any AI attribution in commits
- **NEVER** include "Generated by Claude", "AI-generated", or similar references
- **NEVER** mention Claude, ChatGPT, Copilot, or any AI tool in commit messages
- Write commit messages as if written by a human developer
- Use conventional commit format: `feat:`, `fix:`, `refactor:`, `docs:`, `test:`

### Code Comments Rules
- **NEVER** include comments like "Generated by AI", "Claude suggestion", etc.
- **NEVER** leave TODO comments referencing AI tools
- **NEVER** include placeholder comments like "// AI: explain this"
- Comments should explain **why**, not **what** (code explains what)
- Remove any auto-generated boilerplate comments

### Testing Requirements
- Write unit tests for all business logic (especially routing/arbitrage)
- Write integration tests for API endpoints
- Test edge cases: yield floor boundaries, market failures, node timeouts
- Aim for >80% code coverage on core A²E logic

### Security Standards
- No secrets in code — use environment variables
- Validate all external input
- Sanitize data before database operations
- Use parameterized queries (Prisma handles this)
- Rate limiting on public endpoints
- CORS configured properly

## Tech Stack (Confirmed)
- **Backend:** Node.js + Fastify + TypeScript
- **Database:** PostgreSQL + Prisma
- **Queue:** BullMQ + Redis
- **Real-time:** Socket.io
- **Frontend:** Next.js 14 + React + Vite + Tailwind
- **Orchestration:** Kubernetes + Docker
- **Process Manager:** PM2

## Architecture Notes
- A²E is a **standalone microservice** — does not touch compute.tokenos.ai codebase
- Single integration point: `POST /route` endpoint
- Config-based market enabling — add credentials to config, market goes live
- No third-party monitoring tools (no Grafana/Prometheus) — custom dashboards only

## A²E Core Logic
```
Route to HIGHEST PAYING market for operators (NOT cheapest for buyers)

if (externalRate > internalRate) → route external
else if (internalRate >= yieldFloor) → route internal
else → route internal, enforce yield floor ($0.85/hr)
```
